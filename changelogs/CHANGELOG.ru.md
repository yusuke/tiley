# Журнал изменений

## [Unreleased]

## [3.1.0] - 2026-03-24

### Изменено

- Всплывающие меню-кебаб (…) в списке окон боковой панели заменены на нативные контекстные меню macOS (правый клик)
- Добавлены кнопки действий (Переместить на экран, Закрыть/Завершить, Скрыть другие приложения) рядом с полем поиска боковой панели
- Миниатюры сетки в предустановках макета теперь отражают соотношение сторон рабочей области экрана (без строки меню и Dock), адаптируясь к портретной или альбомной ориентации.

### Исправлено

- Исправлена проблема, при которой изменение размера окна иногда не срабатывало при перемещении на другой экран (особенно на более высокий портретный монитор), благодаря введению механизма повторной попытки при перемещении между экранами

### Удалено

- Удалены всплывающие кнопки меню-кебаб и закрытия из строк боковой панели (заменены контекстными меню и панелью действий)

## [3.0.1] - 2026-03-23

### Добавлено

- При выводе окна на передний план через Enter или двойной клик окно теперь перемещается на экран, где находится указатель мыши, если он отличается. Окно репозиционируется для размещения на экране и изменяет размер только при необходимости.

### Изменено

- Производительность отображения оверлея улучшена примерно на 80% благодаря пулингу/повторному использованию контроллеров, отложенной загрузке списка окон и приоритетному рендерингу целевого экрана
- Внутренняя настройка журнала отладки переименована с `useAppleScriptResize` на `enableDebugLog` для более точного отражения назначения

### Исправлено

- Исправлена проблема, при которой изменение размера окна на основном экране для некоторых приложений (например, Chrome) молча игнорировалось. Механизм повторной попытки с перемещением, используемый для дополнительных экранов, теперь применяется и к основному экрану
- Исправлено: нажатие на значок в строке меню при видимом оверлее теперь закрывает оверлей (как ESC) вместо открытия главного окна

## [3.0.0] - 2026-03-23

### Добавлено

- Интеграция SDK TelemetryDeck для сбора статистики использования с соблюдением конфиденциальности (открытие оверлея, применение макета, применение пресета, изменение настроек)
- Окна в боковой панели группируются по экранам и приложениям; приложения с несколькими окнами отображают заголовок с вложенными строками окон
- Заголовки экранов в боковой панели имеют меню с действиями «Собрать окна» и «Переместить окна на» для управления окнами между экранами
- Меню заголовка приложения с «Переместить все окна на другой экран», «Скрыть остальные» и «Завершить»
- Меню приложений с одним окном с «Переместить на другой экран», «Скрыть остальные» и «Завершить»
- Пустые экраны (без окон) отображаются в боковой панели с заголовком экрана

### Изменено

- Фон сетки теперь точно отражает настройки отображения обоев macOS (заполнение, вписать, растянуть, по центру и мозаика), включая правильное масштабирование плиток, физическое соотношение пикселей для режима по центру и цвет заливки для областей letterbox
- Предпросмотр сетки макета теперь отображает строку меню, Dock и вырез, обеспечивая более точное представление реального экрана

### Исправлено

- Исправлена проблема, при которой окно перемещалось в неожиданную позицию после изменения размера, когда оно уже находилось в целевой позиции. Обход дедупликации AX через предварительное смещение
- Уменьшено мерцание при изменении размера на неосновных экранах. Сначала размер изменяется на месте; переброс на основной экран происходит только при полном отказе
- При перебросе на основной экран окно теперь размещается у нижнего края (почти за пределами экрана) вместо верхнего левого угла, минимизируя мерцание

## [2.2.0] - 2026-03-21

### Изменено

- Невыбранные ячейки сетки теперь прозрачны
- Пропорции сетки теперь соответствуют видимой области экрана (без строки меню и Dock); если сетка оказывается слишком высокой, её ширина пропорционально уменьшается, чтобы оставалось место для не менее 4 пресетов
- Фон сетки макета теперь отображает изображение рабочего стола (полупрозрачное, с закруглёнными углами)
- Ячейки, выбранные перетаскиванием, теперь полупрозрачны — изображение рабочего стола просвечивает сквозь них
- Подсветка при наведении на пресет в сетке теперь использует тот же стиль, что и выбор перетаскиванием

### Добавлено

- Боковая панель списка окон теперь отображается на всех экранах в многомониторных конфигурациях, а не только на целевом экране
- Состояние боковой панели (видимость, выбранный элемент, текст поиска) синхронизируется между всеми окнами на экранах
- Необязательный лог отладки изменения размера (`~/tiley.log`) (Настройки > Отладка)

### Исправлено

- Исправлена проблема, при которой размещение окна использовало устаревшую геометрию экрана, когда Dock или строка меню автоматически показывались/скрывались во время отображения оверлея
- Исправлено изменение размера окон на неосновных экранах в конфигурациях со смешанным DPI; окно временно перемещается на основной дисплей для изменения размера, затем помещается в целевую позицию
- Исправлена проблема, при которой позиция не применялась после изменения размера, когда некоторые приложения молча отменяют изменения позиции (обход дедупликации AX)
- Когда минимальный размер окна приложения не позволяет установить запрошенный размер, позиция окна пересчитывается, чтобы оно оставалось в видимой области экрана
- Устранено видимое мерцание окон при переключении целевого окна между экранами; окна больше не пересоздаются при смене экрана

## [2.1.0] - 2026-03-20

### Добавлено

- Двойной клик по окну в боковой панели выводит его на передний план и закрывает сетку компоновки
- Контекстное меню (кнопка с многоточием) в строках окон боковой панели с тремя действиями:
  - «Закрыть другие окна [Приложения]» — закрывает другие окна того же приложения (отображается только при наличии нескольких окон)
  - «Завершить [Приложение]» — завершает приложение
  - «Скрыть окна кроме [Приложения]» — скрывает все остальные приложения (аналог Cmd-H), отображает выбранное приложение, если оно было скрыто
- Скрытые (Cmd-H) приложения теперь отображаются в боковой панели как записи-заполнители (только имя приложения) с 50% непрозрачностью
- При выборе скрытого приложения (Enter, двойной клик, изменение размера через сетку/компоновку) оно автоматически отображается и операция выполняется над его передним окном
