# 변경 이력

## [Unreleased]

### 변경

- 오버레이 창을 투명도 0으로 미리 렌더링하여 화면에 유지하도록 변경하여, 레이아웃 그리드 표시 시 알파 값만 변경하면 되므로 체감 지연 시간이 크게 감소

## [4.2.1] - 2026-04-04

### 변경

- 크기 조절 도구 모음 버튼에 셰브론 표시를 추가하여 드롭다운 메뉴임을 더 명확하게 표시
- 크기 조절 동작 타이밍을 개선하여 대상 창 크기 조절 전에 Tiley 창이 사라지도록 변경하여 더 직관적인 조작감 제공

## [4.2.0] - 2026-04-04

### 추가

- 도구 모음 버튼 또는 오른쪽 클릭 메뉴에서 창을 사전 정의된 크기(16:9, 16:10, 4:3, 9:16)로 조절하는 기능 추가; 현재 화면을 초과하는 크기는 자동으로 제외
- 크기 조절 메뉴 항목에 마우스를 올리면 대상 화면에 실제 크기 미리보기와 그리드에 미니어처 창 미리보기 표시 (프리셋 레이아웃 호버와 동일한 스타일)
- 그리드 드래그 선택 중 미니어처 창 미리보기(제목 표시줄 및 앱 아이콘 포함) 표시

## [4.1.2] - 2026-04-03

### 추가

- 2개 이상의 윈도우 선택 시 사이드바 윈도우 항목 오른쪽에 선택 순서 인덱스 배지 표시

### 변경

- 사이드바의 윈도우 목록을 워크스페이스 이벤트 리스너(앱 활성화, 실행, 종료)를 통해 백그라운드에서 사전 캐싱하도록 변경. 오버레이를 열면 윈도우 목록이 즉시 표시됨
- 앱별로 그룹화된 사이드바 항목의 하이라이트 동작 개선. 앱 헤더는 모든 윈도우가 선택된 경우에만 선택 상태로 표시되며, 앱 헤더에 마우스를 올리면 헤더와 모든 하위 윈도우가 동시에 하이라이트됨
- 전체 화면 해제 동작 개선: AXFullScreen 속성을 직접 설정하는 방식으로 변경하고(버튼 누르기 폴백 포함), 애니메이션 완료까지 최대 2초 대기하도록 변경

### 수정

- 최전면 애플리케이션에 윈도우가 없을 때 오버레이가 열리지 않던 문제 수정. '윈도우 없음' 메시지를 표시하고 드래그 조작을 비활성화
- Finder의 데스크톱이 크기 조절 가능한 윈도우로 취급되던 문제 수정. 데스크톱에 포커스가 있을 때 Finder의 최전면 실제 윈도우를 대상으로 하며, 윈도우가 없으면 '윈도우 없음'을 표시
- 최전면 애플리케이션에 창이 없는 경우(창이 열리지 않은 Finder, 메뉴 바 전용 앱 등) 오버레이가 열리지 않는 문제 수정. 화면에서 가장 앞에 있는 가시 창을 대체 대상으로 사용하도록 변경
- 일부 앱(예: Notion)에서 비주 디스플레이의 창 위치가 올바르게 적용되지 않는 문제를 수정. 크기 변경 후 앱이 비동기적으로 위치를 되돌리는 경우를 처리하기 위해 위치 검증 및 재시도 로직 추가

## [4.1.1] - 2026-03-31

### 변경

- 다음 윈도우 선택 기본 단축키를 Tab에서 Space로, 이전 윈도우를 Shift+Tab에서 Shift+Space로 변경
- 오버레이를 닫을 때 대피된 윈도우가 항상 애니메이션과 함께 원래 위치로 복귀하도록 변경

## [4.1.0] - 2026-03-31

### 추가

- 수정자 키를 누른 채 윈도우 전환 기능 추가 (Cmd+Tab 스타일): 오버레이를 연 후 토글 수정자 키를 누른 채 트리거 키를 반복 누르면 윈도우가 순차 전환되며, 수정자 키를 놓으면 선택된 윈도우가 최전면으로 이동. 수정자 키를 누른 채 레이아웃 로컬 단축키를 누르면 해당 레이아웃 적용
- 설정 화면에 서드파티 라이선스 표시 섹션 추가 (Sparkle, TelemetryDeck)

### 변경

- 설정 및 권한 패널을 별도의 일반(비플로팅) 레벨 윈도우로 분리하여 Sparkle 업데이트 대화상자 및 기타 OS 윈도우가 위에 표시되도록 변경
- 사이드바를 항상 표시하도록 변경하고 사이드바 표시/숨기기 토글 버튼 제거
- 설정 버튼을 푸터 바에서 사이드바 액션 바 왼쪽 끝으로 이동
- 미니 스크린 프리뷰의 모서리를 디스플레이 유형에 관계없이 네 모서리 모두 둥글게 처리
- 미니어처 윈도우 타이틀 바에 윈도우 제목과 함께 앱 이름 표시
- "업데이트 사용 가능" 배지를 설정 버튼의 빨간 점과 툴팁으로 대체; 설정 패널에서는 "업데이트 확인" 버튼에 팝오버로 표시

## [4.0.9] - 2026-03-30

### 수정

- 특정 앱에서 윈도우 크기 조정이 실패하고 위치가 어긋나는 문제 수정: 초기 크기 조정이 무시되었을 때 바운스 위치가 화면 하단(확장 여지 없음)이어서 윈도우가 잘못된 위치에 남겨졌음. 바운스 위치를 가시 영역 상단으로 변경하고 크기 조정 실패 시 위치를 명시적으로 복원
- 배경 윈도우를 선택한 후 대피된 윈도우가 원래 위치로 복원되지 않는 경우가 있는 문제 수정: 복원 시 오래된 목록에서 윈도우를 검색했기 때문에 실패할 수 있었음. 대피 추적 데이터에 윈도우 참조를 직접 저장하고, 복원 애니메이션이 완료될 때까지 정리를 지연하도록 개선
- "단축키 추가" / "전역 단축키 추가" 버튼이 중앙 근처 클릭에만 반응하던 문제 수정: 패딩과 배경을 버튼 레이블 내부로 이동하여 표시 영역 전체를 클릭 가능하도록 변경

## [4.0.8] - 2026-03-30

### 수정

- 접근성 권한 요청 중 권한 패널 창이 다른 앱 및 OS 다이얼로그 위에 표시되지 않도록 수정
- macOS Tahoe 26.4에서 배경화면 미리보기가 표시되지 않는 문제 수정: 배경화면 Store plist 구조 변경(`Desktop` → `Linked` 키) 대응, 사진 배경화면은 배경화면 에이전트 BMP 캐시에서 로드, `FillScreen` 배치 값(Tahoe에서 `Stretch` 대체) 추가, 비시스템 배경화면 제공자의 표시 모드 설정 활성화
- 이미지의 DPI 메타데이터가 72가 아닌 경우(예: 144 DPI Retina 스크린샷) 중앙 및 타일 배경화면 표시 모드에서 이미지가 너무 작게 렌더링되는 문제 수정. 항상 실제 픽셀 크기를 사용하도록 변경

## [4.0.7] - 2026-03-29

### 수정

- 타일 배경화면 표시 모드가 미니 화면 미리보기에 반영되지 않던 문제 수정 (macOS 배경화면 Store plist의 배치 값 "Tiled"가 올바르게 매칭되지 않았음)
- 배경화면 표시 문제 진단을 위해 배경화면 해석 파이프라인에 디버그 로그 추가

## [4.0.6] - 2026-03-29

### 추가

- 멀티 레이아웃 프리셋에 호버 시 미니 화면 그리드, 실제 크기 미리보기, 사이드바 창 목록에 레이아웃 인덱스 번호를 표시하여 색각 특성에 관계없이 어떤 레이아웃이 적용되는지 직관적으로 파악 가능

### 변경

- macOS Tahoe의 L&F에 맞게 설정 화면 UI 조정: 툴바·액션 바 버튼을 캡슐 형태 및 시스템 적응형 호버/프레스 배경으로 통일, 설정 섹션 카드를 테두리 없는 연한 회색 배경으로 변경, 토글 크기를 시스템 설정과 동일하게 축소, 단축키 목록을 재구성하고 "디스플레이 이동 단축키" 섹션 독립

### 수정

- 프리셋 레이아웃 수를 초과하는 사이드바 창이 기본 선택 색상 대신 마지막 레이아웃 색상으로 올바르게 표시되도록 수정

## [4.0.5] - 2026-03-29

### 수정

- 선택한 대상 창을 표시하기 위해 이동된 창이 빠른 순환 중에도 원래 위치로 올바르게 복원되도록 수정
- 단일 창 크기 조정 미리보기가 다중 창 레이아웃 미리보기보다 흐리게 표시되던 문제를 수정하여 동일한 불투명도를 사용하도록 변경

## [4.0.4] - 2026-03-29

### 추가

- 프리셋 호버 시 미니 스크린 레이아웃 프리뷰에 창 타이틀 바(앱 아이콘, 앱 이름, 창 제목) 표시

### 변경

- 실물 크기 레이아웃 프리뷰의 타이틀 바에 앱 이름과 창 제목을 함께 표시하도록 변경 ("앱 이름 — 창 제목" 형식)

## [4.0.3] - 2026-03-29

### 추가

- 다중 레이아웃 프리셋에서 창을 하나만 선택해도 실제 Z-order(최전면부터 순서대로)로 여러 창을 리사이즈
- 선택된 창 수가 레이아웃 정의 수보다 적을 때, 선택된 창을 항상 프라이머리로 처리하고 나머지는 Z-order로 보충
- 프리셋 호버 시 사이드바의 대상 창 행을 레이아웃 색상(파랑, 초록, 주황, 보라)으로 하이라이트 표시

## [4.0.2] - 2026-03-29

### 변경

- 실제 크기 레이아웃 미리보기에서 프리셋에 정의된 선택 수만큼만 미리보기를 표시하도록 변경 (선택 수를 초과하는 창의 미리보기는 표시하지 않음)

## [4.0.1] - 2026-03-29

### 변경

- 선택 색상 팔레트를 파랑, 초록, 주황, 보라 4색 순환으로 변경하여 5번째 선택이 1번째와 동일한 색상으로 표시
- 기본 프리셋(왼쪽/오른쪽/위/아래 절반)에 보조 선택으로 반대쪽 절반 추가

## [4.0.0] - 2026-03-29

### 추가

- 다중 선택 레이아웃 프리셋: 하나의 프리셋에 여러 그리드 영역을 정의하여 다른 창을 다른 위치에 배치 가능
  - 프리셋 편집기에서 드래그할 때마다 새 선택 영역 추가 (1번째, 2번째, 3번째...)
  - 각 선택 영역에 인덱스 번호와 삭제 버튼 표시
  - 선택 영역 간 겹침 방지 (시각적 피드백 포함)
  - 다중 선택 프리셋 적용 시 선택 순서에 따라 창 할당: 처음 선택한 창이 선택 1, 다음 선택한 창이 선택 2 등
  - 프리셋 썸네일과 실제 크기 미리보기에 모든 선택 영역을 인덱스 색상으로 표시
  - 그리드 선택 영역은 화면 가장자리에서 1pt 여백으로 가시성 향상

### 변경

- 다중 창 순서가 사이드바 Z순서에서 선택 순서로 변경
  - 처음 선택한 창이 항상 프라이머리, Cmd+클릭으로 추가한 창은 추가 순서대로 배치
  - Shift+클릭 범위 선택에서는 앵커 창이 프라이머리 유지
  - 레이아웃 프리셋 적용, 최전면화(Enter), 미리보기 표시에 영향

## [3.4.0] - 2026-03-28

### 추가

- 사이드바에서 여러 윈도우를 선택하여 일괄 작업 수행 가능
    - 앱 헤더를 클릭하면 해당 앱의 모든 윈도우 선택
    - Cmd+클릭으로 개별 윈도우 추가/해제
    - Shift+클릭으로 사이드바에서 연속된 윈도우 범위 선택
- 다중 선택 시 일괄 작업: 최전면으로(사이드바 순서의 Z-order 유지), 그리드로 리사이즈/이동, 디스플레이 이동, 닫기/종료
- 여러 윈도우를 닫을 때 모든 윈도우가 선택된 앱은 앱 자체를 종료(Finder 제외)

### 변경

- 사이드바에서 앱 헤더를 클릭하면 해당 앱의 모든 윈도우를 선택하도록 변경(이전에는 최전면 윈도우만 선택)
- 앱 그룹 내 윈도우를 선택할 때 애플리케이션 헤더도 하이라이트되도록 변경
- Finder가 아닌 앱이 여러 윈도우를 가진 경우, 액션 바에 "윈도우 닫기" 버튼 옆에 "앱 종료" 버튼을 추가 표시
- "윈도우 닫기" 툴팁에 윈도우 이름을 표시하도록 변경 (예: "문서" 닫기)

## [3.3.2] - 2026-03-28

### 추가

- 다음 창 선택, 이전 창 선택, 최전면으로 가져오기, 닫기/종료 단축키를 환경설정의 단축키 섹션에서 설정할 수 있게 됨
- 사이드바에서 창을 오른쪽 클릭할 때 "[앱 이름]의 다른 윈도우 닫기" 컨텍스트 메뉴 항목 추가 (앱에 여러 창이 있을 때만 표시)

### 변경

- 단축키 설정 화면을 창 동작 단축키와 디스플레이 이동 단축키 두 그룹으로 재구성함
- 디스플레이 이동 단축키를 글로벌 전용으로 변경하고, 로컬 단축키 지원 및 설정 항목을 제거함
